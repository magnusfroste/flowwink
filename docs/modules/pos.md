# POS — Point of Sale

In-store / counter sales register that integrates with FlowWink's commerce ledger.

## What it does

Lets staff:
- Open a cashier shift on a register with opening cash
- Ring up sales with line items, tax, multiple payment methods
- Print/email receipts
- Close the shift with cash counting + auto-computed variance

## Schema

| Table | Purpose |
|-------|---------|
| `pos_registers` | Physical or virtual cash registers (location, currency, default tax rate) |
| `pos_sessions` | Open/close shifts per register and cashier — opening cash, closing cash, expected vs actual cash variance |
| `pos_sales` | Completed sale receipts — totals, payment method, customer link, status (completed / refunded / voided) |
| `pos_sale_lines` | Line items per sale (product, qty, unit price, discount, tax) |

Receipt numbers are generated by the `pos_receipt_seq` sequence: `POS-000001`, `POS-000002`, …

## RPCs (SECURITY DEFINER)

- `open_pos_session(register_id, opening_cash_cents, cashier_name)` — fails if register already has an open session.
- `close_pos_session(session_id, closing_cash_cents)` — computes expected = opening + cash sales, returns variance.
- `record_pos_sale(register_id, session_id, lines[], payment_method, customer_email, discount_cents)` — atomically creates sale + lines, computes subtotal/tax/total, updates session counters, emits `pos.sale_completed` event.

## Skills (MCP-exposed)

- `open_pos_session`
- `close_pos_session`
- `record_pos_sale`
- `list_pos_sales` (generic CRUD via `agent-execute`)

## Routes

- `/admin/pos` — register, session and history tabs

## Events emitted

- `pos.sale_completed` — fan-out to FlowPilot for daily summaries, low-stock alerts, etc.
